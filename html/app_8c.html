Sets up LETIMER0 for specified PWM and interrupt operation.

This function uses aspects of the app_letimer_pwm_struct in order to specify specific operations, like PWM period for the LETIMER0, as well as, interrupt functionallity that will only be used in this specific application.

Note
This function is called once in app_peripheral_setup() in order to setup LETIMER0 in the operation mode specific to this application.
Parameters
[in]periodDesired total period for PWM operation in seconds
[in]act_periodDesired active period for PWM operation in seconds (how long signal should be on)
[in]out0_routeLocation 0 for the generated PWM to be routed to (ex. location 16 or 17 for LEDs)
[in]out1_routeLocation 1 for the generated PWM to be routed to (ex. location 16 or 17 for LEDs)
[in]comp0_cbUsed to set the event scheduler when comp0 triggers a callback
[in]comp1_cbUsed to set the event scheduler when comp1 triggers a callback
[in]underflow_cbUsed to set the event scheduler when underflow triggers a callback

◆ app_peripheral_setup()

void app_peripheral_setup ( void  )

This function initializes/opens all of our peripherals.

This function calls our drivers for the CMU, GPIO and letimer, in order to initialize each peripheral. Additionally, this function will initialize our event scheduler and sleep driver. It sets up LETIMER0 with a specified PWM, then starts the timer.

Note
This function will be called in main.c in order to set everything up for operation before we start operation.

◆ scheduled_si1133_read_cb()

void scheduled_si1133_read_cb ( void  )

Call back function that is called once white light read operation is completed for si1133.

This function handles operation that should occur after a successful i2c white light read operation of the si1133.

Note
This function retrieves the value read from the si1133 peripheral and turns on BLUE LED if read value is less than expected value or turns off if value is greater than or equal to expected value. Transmits data values through bluetooth module.
